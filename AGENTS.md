# NanaBot AGENTS Rules

本文件定义 NanaBot 项目的全局协作规范。所有后续改动默认必须遵守。

## 1. 强制规则（Hard Rules）

1. 注释必须使用中文。输出给用户的信息也必须使用中文。
2. 文档必须中英文同步更新。
   - 需要同步的文档至少包含 `README.md` 与 `README.zh-CN.md`。
3. 新增命令后必须同步更新 README 的命令说明。
   - 需要明确区分普通用户命令与管理员命令。
4. 方法注释需要说明参数含义与允许值，除非参数语义显而易见。
   - 示例：`add_to_whitelist(target_type, target_id)` 需要说明 `target_type` 仅允许 `group` 或 `user`。
5. 所有对 `.env` 的修改都必须同步到 `.env.example`，并且必须脱敏。
   - `.env.example` 只能保留模板值，不允许包含真实 token、密钥、账号等敏感信息。
6. 所有在 `data/config.json` 中新增的配置项都必须同步在 README 中进行说明。
7. 小范围改动必须优先使用局部修改，禁止为了少量文字调整而无必要地整文件删除后重写。
    - 只有在文件存在明显编码损坏、结构严重混乱、或局部修改成本显著高于整体整理时，才允许整文件重写。
    - 若确实需要整文件重写，必须在执行前说明原因。

## 2. 代码与注释规范（Code Style）

1. 新增模块、函数、方法时优先补充简洁中文注释，说明“做什么、输入是什么、返回是什么”。
2. 业务关键分支，例如权限校验、白名单拦截、限流拒绝等，必须写中文注释说明原因。
3. 注释要求准确、简短、可执行，禁止空泛描述。

## 3. 文档同步规范（Docs Sync）

1. 任何面向用户的行为变化都需要同步 README 双语文档，包括但不限于：
   - 命令名称或命令参数变更
   - 配置项新增、删除、默认值变化
   - 权限模型变化，例如普通用户与管理员权限
2. README 的中英文结构应保持一致，避免只更新单边文档。
3. README 小改动应优先做最小范围补丁，不要无意义改写整份文档。

## 4. 配置与安全规范（Config & Security）

1. 新增配置项时必须同时更新：
    - 实际配置文件，例如 `.env`
    - 模板配置文件 `.env.example`，且必须使用脱敏后的占位值
   - 双语 README 的“配置说明”
2. 涉及权限和安全的改动，例如白名单、管理员、限流，需要在代码中保留清晰中文注释。
3. 禁止将真实敏感信息，例如真实 token、密钥、密码，写入示例文档。

## 5. 交付前自检（Pre-Delivery Checklist）

提交修改前至少检查以下项目：

1. 新增命令是否已经写入 `README.md` 与 `README.zh-CN.md`。
2. 管理员命令是否在文档中明确标注权限。
3. 新增或修改的方法是否具备必要中文注释与参数说明。
4. 配置项变更是否同步到文档。
5. 是否采用了与改动规模相匹配的编辑方式，避免无必要整文件重写。
6. 代码是否能通过最小运行验证，至少保证导入和启动不因语法错误失败。
